# Integration notes for FrappeHR

Open source human resource management platform, available at [GitHub project](https://github.com/frappe/hrms).  Frappe includes a [REST API](https://docs.frappe.io/framework/user/en/guides/integration/rest_api) .


## Authentication

Service account tokens are used to authenticate:

``` bash
FRAPPE_HOST=frappe.example.com
FRAPPE_PORT=8000
API_KEY=abc123
API_SECRET=xyz987

curl -H "Authorization: token $API_KEY:$API_SECRET" http://$FRAPPE_HOST:$FRAPPE_PORT/api/method/frappe.auth.get_logged_user
```

## Employees

Employee records have the following attributes:

| Attribute              | Description                                                              | Notes                                              |
|------------------------|--------------------------------------------------------------------------|----------------------------------------------------|
| name                   | Unique identifier, e.g. HR\_EMP\_0001                                      | Generated by Frappe                                |
| employee               | Synonym for _name_                                                       |                                                    |
| first\_name             | First name                                                               | Required                                           |
| last\_name              | Last name                                                                | Required                                           |
| employee\_name          | Full name, e.g. _Joe Bloggs_                                             | Generated by Frappe                                |
| gender                 | Gender.  "Prefer not to say" is a valid value                            | Required                                           |
| date\_of\_birth          | Date of birth.  "1900-01-01" is a valid value if actual date is unknown. | Required                                           |
| date\_of\_joining        | First day of work.                                                       | Required                                           |
| status                 | One of "Active", "Inactive", "Suspended", "Left".                        | "Active" is the default when creating new records. |
| reports\_to            | Unique identifer of employee's manager                       |  |
| prefered\_contact\_email | Email used by Frappe to send notifications to Employee                   |                                                    |



Fetch a list of all employee IDs with the _api/resource/Employee_ end point:

``` bash

curl -H "Authorization: token $API_KEY:$API_SECRET" http://$FRAPPE_HOST:$FRAPPE_PORT/api/resource/Employee
```

Fetch attributes of one Employee by referencing the employee's ID:

``` bash

curl -H "Authorization: token $API_KEY:$API_SECRET" http://$FRAPPE_HOST:$FRAPPE_PORT/api/resource/Employee/HR-EMP-001
```

Create an Employee record via a POST to the _api/resource/Employee_ end point.

Mandatory attributes include:
 
- first\_name
- last\_name
- gender
- date\_of\_birth
- date\_of\_joining

``` bash

curl -X POST \
     -H "Authorization: token $API_KEY:$API_SECRET" \
     -d '{"first_name":"Joe", "last_name": "Bloggs", "gender": "Other", "date_of_birth": "2000-01-01", "date_of_joining": "2025-11-17"}' \
      http://$FRAPPE_HOST:$FRAPPE_PORT/api/resource/Employee
```
